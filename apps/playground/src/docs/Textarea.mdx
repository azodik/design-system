# Textarea Component

The Textarea component provides multi-line text input for longer content and messages. It's perfect for collecting detailed user input and comments.

## Overview

Textarea components are used for collecting multi-line text input from users. They provide a larger input area for longer content like comments, descriptions, or messages.

### When to use
- Collecting longer text input
- Comment forms and feedback
- Description fields
- Message composition
- Multi-line content entry

### When not to use
- For single-line text (use Input instead)
- For rich text editing (use rich text editor)
- For code editing (use code editor)
- When space is very limited

## With Validation

<CodeBlockWithPreview 
  title="Textarea with Validation"
  preview={
    <div>
      <label htmlFor="validated">Message</label>
      <Textarea
        id="validated"
        value=""
        onChange={() => console.log('Text changed')}
        error="Message must be at least 10 characters"
        placeholder="Enter your message (10-500 characters)..."
        rows={4}
      />
      <div className="text-sm text-gray-500 mt-1">
        0/500 characters
      </div>
    </div>
  }
>
{`import { Textarea } from '@azodik/ui';
import { useState } from 'react';

export const ValidatedTextareaExample = () => {
  const [value, setValue] = useState('');
  const [error, setError] = useState('');

  const handleChange = (e) => {
    const newValue = e.target.value;
    setValue(newValue);
    
    if (newValue.length < 10) {
      setError('Message must be at least 10 characters');
    } else if (newValue.length > 500) {
      setError('Message must be less than 500 characters');
    } else {
      setError('');
    }
  };

  return (
    <div>
      <label htmlFor="validated">Message</label>
      <Textarea
        id="validated"
        value={value}
        onChange={handleChange}
        error={error}
        placeholder="Enter your message (10-500 characters)..."
        rows={4}
      />
      <div className="text-sm text-gray-500 mt-1">
        {value.length}/500 characters
      </div>
    </div>
  );
};`}
</CodeBlockWithPreview>

## With Character Count

<CodeBlockWithPreview 
  title="Textarea with Character Count"
  preview={
    <div>
      <label htmlFor="character-count">Tweet</label>
      <Textarea
        id="character-count"
        value=""
        onChange={() => console.log('Text changed')}
        placeholder="What's happening?"
        rows={3}
        maxLength={280}
      />
      <div className="text-sm mt-1 text-gray-500">
        280 characters remaining
      </div>
    </div>
  }
>
{`import { Textarea } from '@azodik/ui';
import { useState } from 'react';

export const CharacterCountTextareaExample = () => {
  const [value, setValue] = useState('');
  const maxLength = 280;

  const handleChange = (e) => {
    setValue(e.target.value);
  };

  const remaining = maxLength - value.length;
  const isOverLimit = remaining < 0;

  return (
    <div>
      <label htmlFor="character-count">Tweet</label>
      <Textarea
        id="character-count"
        value={value}
        onChange={handleChange}
        placeholder="What's happening?"
        rows={3}
        maxLength={maxLength}
        className={isOverLimit ? 'border-red-500' : ''}
      />
      <div className={'text-sm mt-1 ' + (isOverLimit ? 'text-red-500' : 'text-gray-500')}>
        {remaining} characters remaining
      </div>
    </div>
  );
};`}
</CodeBlockWithPreview>

## Accessibility

- Textareas have proper labels and associations
- Error messages are announced to screen readers
- Focus management is handled correctly
- Keyboard navigation is supported
- Color contrast meets WCAG guidelines

## Design Guidelines

- Always provide clear labels
- Use appropriate placeholder text
- Show character limits when applicable
- Provide clear validation feedback
- Ensure sufficient touch targets on mobile
