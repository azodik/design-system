# Button Component

The Button component provides interactive elements for user actions and navigation. It supports multiple variants, sizes, and states to fit different use cases.

## Overview

Buttons are the primary interactive elements in your application. They allow users to perform actions, submit forms, and navigate through your interface.

### When to use
- Primary actions and calls-to-action
- Form submissions
- Navigation between pages or sections
- Confirming or canceling actions
- Triggering modals or dropdowns

## Accessibility

- Buttons have proper focus states
- Screen readers can identify button purpose
- Disabled buttons are properly announced
- Keyboard navigation is supported
- Color contrast meets WCAG guidelines

## Design Guidelines

- Use primary buttons for main actions
- Use secondary buttons for secondary actions
- Use destructive buttons for dangerous actions
- Keep button text concise and action-oriented
- Ensure sufficient touch targets on mobile devices
