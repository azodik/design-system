# Checkbox Component

The Checkbox component provides binary choice controls for enabling or disabling options. It's perfect for forms that require multiple selections or toggles.

## Overview

Checkboxes allow users to select one or more options from a set of choices. They provide clear visual feedback for selected states and are essential for forms and settings.

### When to use
- Multiple selections from a list
- Enabling or disabling features
- Agreeing to terms and conditions
- Filtering options
- Settings and preferences

### When not to use
- For single choice selection (use Radio instead)
- For primary actions (use Button instead)
- When you need immediate action (use Switch instead)
- For navigation (use other navigation components)

<CodeBlockWithPreview

import { useState } from 'react';

export const ControlledCheckboxExample = () => {

    setIsChecked(e.target.checked);

        id="newsletter"
        checked={isChecked}
        onChange={handleChange}
      />

};`}

## Multiple Checkboxes

<CodeBlock title="Multiple Checkboxes">
{`import { Checkbox } from '@azodik/ui';
import { useState } from 'react';

export const MultipleCheckboxesExample = () {
  const [selectedItems, setSelectedItems] = useState([]);

  const items = [
    { id: 'item1', label: 'Option 1' },
    { id: 'item2', label: 'Option 2' },
    { id: 'item3', label: 'Option 3' },
    { id: 'item4', label: 'Option 4' }
  ];

  const handleChange = (itemId, checked) => {
    if (checked) {
      setSelectedItems([...selectedItems, itemId]);
    } else {
      setSelectedItems(selectedItems.filter(id => id !== itemId));
    }
  };

  return (
    <div className="space-y-2">
      <h3>Select your preferences:</h3>
      {items.map(item => (
        <div key={item.id} className="flex items-center space-x-2">
          <Checkbox
            id={item.id}
            checked={selectedItems.includes(item.id)}
            onChange={(e) => handleChange(item.id, e.target.checked)}
          />
          <label htmlFor={item.id}>{item.label}</label>
        </div>
      ))}
      <p className="text-sm text-gray-500">
        Selected: {selectedItems.length} items
      </p>
    </div>
  );
}`}
</CodeBlock>

## With Indeterminate State

<CodeBlock title="Indeterminate Checkbox">
{`import { Checkbox } from '@azodik/ui';
import { useState } from 'react';

export const IndeterminateCheckboxExample = () {
  const [items, setItems] = useState([
    { id: 'item1', label: 'Item 1', checked: false },
    { id: 'item2', label: 'Item 2', checked: false },
    { id: 'item3', label: 'Item 3', checked: false }
  ]);

  const allChecked = items.every(item => item.checked);
  const someChecked = items.some(item => item.checked);
  const indeterminate = someChecked && !allChecked;

  const handleSelectAll = (checked) => {
    setItems(items.map(item => ({ ...item, checked })));
  };

  const handleItemChange = (itemId, checked) => {
    setItems(items.map(item => 
      item.id === itemId ? { ...item, checked } : item
    ));
  };

  return (
    <div className="space-y-2">
      <div className="flex items-center space-x-2">
        <Checkbox
          id="select-all"
          checked={allChecked}
          indeterminate={indeterminate}
          onChange={(e) => handleSelectAll(e.target.checked)}
        />
        <label htmlFor="select-all">Select All</label>
      </div>
      
      {items.map(item => (
        <div key={item.id} className="flex items-center space-x-2 ml-4">
          <Checkbox
            id={item.id}
            checked={item.checked}
            onChange={(e) => handleItemChange(item.id, e.target.checked)}
          />
          <label htmlFor={item.id}>{item.label}</label>
        </div>
      ))}
    </div>
  );
}`}
</CodeBlock>

## Accessibility

- Checkboxes have proper labels and associations
- Screen readers can identify checkbox states
- Keyboard navigation is supported
- Focus management is handled correctly
- Proper ARIA attributes are applied

## Design Guidelines

- Always provide clear labels
- Use consistent sizing across your application
- Provide clear visual feedback for all states
- Group related checkboxes together
- Ensure sufficient touch targets on mobile
