export const ValidatedInputPreview = () => {
  const [value, setValue] = useState('');
  const [error, setError] = useState('Must be at least 3 characters');

  const handleChange = (e) => {
    const newValue = e.target.value;
    setValue(newValue);
    
    if (newValue.length < 3) {
      setError('Must be at least 3 characters');
    } else {
      setError('');
    }
  };

  return (
    <Input 
      label="Validated Input"
      value={value}
      onChange={handleChange}
      error={error}
      placeholder="Enter at least 3 characters"
    />
  );
};

# Input Component

The Input component provides text input fields for user data entry with various types, validation states, and interactive features.

## Introduction

The Input component includes several complementary features to handle various text input and data entry scenarios:

- Input: the main wrapper for creating text input fields with different types and validation states.
- Input Types: support for various input types (text, email, password, number, tel, url) with appropriate validation.
- Input Validation: built-in validation states with error messages, success indicators, and real-time feedback.
- Input Accessibility: proper labels, help text, error announcements, and keyboard navigation support.
- Input Customization: flexible sizing, styling, placeholder text, and integration with form libraries.
- Input States: support for disabled, readonly, and interactive states with appropriate visual feedback.

### When to use
- Collecting text input from users
- Form fields for data entry
- Search inputs
- Email, password, and other specific input types
- User registration and login forms

### When not to use
- For multi-line text (use Textarea instead)
- For selecting from options (use Select instead)
- For binary choices (use Checkbox or Radio instead)

## Accessibility

- Inputs have proper labels and associations
- Error messages are announced to screen readers
- Focus management is handled correctly
- Keyboard navigation is supported
- Color contrast meets WCAG guidelines

## Design Guidelines

- Always provide clear labels
- Use appropriate input types for better UX
- Provide helpful placeholder text
- Show validation errors clearly
- Ensure sufficient touch targets on mobile
