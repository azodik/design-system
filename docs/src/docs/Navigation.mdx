# Navigation Component

The Navigation component provides horizontal navigation bars for site-wide navigation. It helps users navigate between different sections and pages of your application.

## Introduction

The Navigation component includes several complementary features to handle various site navigation and user flow scenarios:

- Navigation: the main wrapper for creating horizontal navigation bars with brand elements and menu items.
- Navigation Items: individual menu items with active states, links, and customizable styling for different sections.
- Navigation Branding: flexible brand/logo area with customizable content and positioning options.
- Navigation Responsiveness: mobile-friendly navigation with collapsible menus and touch-optimized interactions.
- Navigation States: active page indication, hover effects, and visual feedback for current location.
- Navigation Accessibility: proper ARIA attributes, keyboard navigation, and screen reader support.

### When to use
- Main site navigation
- Section navigation within an app
- Primary navigation menus
- Header navigation bars
- Top-level page navigation

## Accessibility

- Navigation has proper semantic structure
- Screen readers can navigate menu items
- Keyboard navigation is supported
- Focus management is handled correctly
- Proper ARIA attributes are applied

## Design Guidelines

- Use clear and descriptive navigation labels
- Keep the number of top-level items reasonable
- Provide clear visual feedback for active states
- Ensure sufficient touch targets on mobile
- Use consistent styling across your application
