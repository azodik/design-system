# Alert Component

The Alert component is used to display important messages to users. It provides different variants for various types of notifications and can include actions like close buttons.

## Introduction

The Alert component includes several complementary features to handle various notification scenarios:

- Alert: the main wrapper for displaying important messages to users.
- Alert Variants: different visual styles (success, warning, error, info) to indicate the type of message.
- Alert Actions: optional close button functionality for dismissible alerts.
- Alert Customization: flexible sizing and styling options for different use cases.
- Alert Icons: automatic icon generation based on variant or custom icon support.
- Alert Accessibility: built-in ARIA attributes and screen reader support for better user experience.

### When to use
- Display success, warning, error, or informational messages
- Show important notifications that require user attention
- Provide feedback for user actions

## With Close Button

<CodeBlockWithPreview 
  title="Dismissible Alert"
  preview={
    <Alert 
      variant="warning" 
      title="Warning" 
      onClose={() => console.log('Alert closed')}
    >
      This alert can be dismissed.
    </Alert>
  }
>
{`import { Alert } from '@azodik/ui';

export default function DismissibleAlert() {
  const handleClose = () => {
    console.log('Alert closed');
  };

  return (
    <Alert 
      variant="warning" 
      title="Warning" 
      onClose={handleClose}
    >
      This alert can be dismissed.
    </Alert>
  );
}`}
</CodeBlockWithPreview>

## Accessibility

- Alerts have appropriate ARIA roles and attributes
- Screen readers will announce alert content
- Focus management is handled for dismissible alerts
- Color is not the only way to convey information

## Design Guidelines

- Use success alerts for positive feedback
- Use warning alerts for important information that doesn't prevent action
- Use error alerts for problems that need immediate attention
- Use info alerts for general information
- Keep alert messages concise and actionable
- Provide clear close actions when appropriate
